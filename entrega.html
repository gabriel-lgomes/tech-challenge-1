1. Descrição do Desafio

Uma rede de hospitais e centros de saúde especializados no atendimento à mulher busca implementar um sistema inteligente de suporte ao diagnóstico e detecção de riscos, capaz de ajudar profissionais de saúde na identificação precoce de condições que afetam a segurança e saúde feminina.

Nesta primeira fase, o objetivo é criar a base do sistema com foco em Machine Learning, permitindo que dados médicos sejam analisados automaticamente para identificar padrões de risco relacionados à saúde das mulheres.

3. Análise Exploratória (EDA)

Notebook: notebooks/01_eda.ipynb

Principais achados

  • O dataset está limpo — sem valores ausentes ou duplicatas, exigindo pré-processamento mínimo.
  • Leve desbalanceamento de classes (~63% benigno / ~37% maligno) — considerado na escolha das métricas.
  • As features de concavidade, perímetro e raio apresentam maior separação entre classes.
  • Alta multicolinearidade entre área, raio e perímetro — tratada na etapa de pré-processamento.
  • Tumores malignos tendem a ser maiores, mais irregulares e com maior concavidade, consistente com a literatura clínica.
Gráficos gerados: distribuição de classes, histogramas por feature, boxplots por diagnóstico, matriz de correlação, ranking de correlação com o target e pairplot das top 5 features — disponíveis em reports/figures/.

5. Modelagem

Notebook: notebooks/03_modeling.ipynb · Módulo: src/evaluation.py

Modelos treinados

ModeloJustificativa
Regressão Logística Baseline linear, interpretável e rápido. Permite verificar se a separação entre classes é linearmente possível.
Random Forest Ensemble de árvores de decisão que captura relações não-lineares. Gera feature importance nativamente e é robusto a outliers.

Por que Recall é a métrica principal?

Falso Negativo = tumor maligno classificado como benigno → paciente vai para casa sem tratamento. Consequência clínica gravíssima.
Falso Positivo = tumor benigno classificado como maligno → exames adicionais desnecessários. Indesejável, mas manejável.

O Recall mede a capacidade do modelo de identificar todos os casos malignos. Deve ser maximizado neste contexto.

Validação cruzada (5-fold)

Antes da avaliação final, ambos os modelos foram validados com cross-validation em 5 folds para verificar estabilidade e ausência de overfitting.

6. Explicabilidade

Feature Importance (Random Forest)

O Random Forest calcula a importância de cada feature com base em quanto ela contribuiu para reduzir a impureza nas árvores. As features de concave points, radius e perimeter (nas estatísticas worst e mean) se destacaram como as mais relevantes para o diagnóstico.

SHAP (SHapley Additive exPlanations)

Além da importância global, o SHAP permite explicar cada predição individualmente — mostrando o quanto cada feature empurrou o resultado para maligno ou benigno em um caso específico. Foram gerados:

  • Beeswarm plot: impacto de todas as features em todas as amostras simultaneamente
  • Bar plot: importância média SHAP (mais robusta que a feature importance simples)
  • Waterfall plot: explicação detalhada de um caso maligno individual

7. Extra — Diagnóstico por Imagem com CNN

Notebook: notebooks/04_cnn.ipynb

Objetivo

Além dos modelos com dados estruturados, foi implementada a parte extra do desafio: uma Rede Neural Convolucional (CNN) para classificação de imagens de ultrassonografia de mama utilizando Transfer Learning.

Arquitetura — VGG16 com Transfer Learning

Com apenas ~546 imagens de treino, treinar uma CNN do zero levaria a overfitting severo. A estratégia adotada foi o Transfer Learning com o VGG16 pré-treinado no ImageNet (1,4 milhão de imagens).

ComponenteDescrição
BaseVGG16 pré-treinado no ImageNet (congelado na fase 1)
Cabeça customizadaGlobalAveragePooling2D → Dense(256, ReLU) → BatchNorm → Dropout(0.5) → Dense(3, Softmax)
Fase 1Treino da cabeça — LR = 1e-3, até 25 épocas com EarlyStopping
Fase 2Fine-tuning das últimas 4 camadas VGG16 — LR = 1e-5 (evita catastrophic forgetting)
AugmentationFlip horizontal, rotação ±10°, zoom ±10% — aplicado apenas no treino

Split

70% treino / 15% validação / 15% teste, estratificado por classe.

Resultados

Os valores exatos são gerados na execução de 04_cnn.ipynb. Gráficos gerados: amostras por classe, curvas de aprendizado (fases 1 e 2), matriz de confusão e predições individuais com percentual de confiança.

Complementaridade com o modelo ML

Os dois sistemas são complementares: a CNN captura informação espacial e textural das imagens de ultrassom, enquanto o Random Forest captura relações quantitativas das biópsias. Um sistema clínico completo combinaria ambos em um modelo ensemble.

8. Discussão Crítica

O modelo pode ser usado na prática?

Sim — como ferramenta de apoio à decisão médica, nunca como diagnóstico autônomo.

O desempenho no dataset Wisconsin é excelente (AUC-ROC próximo de 1,0), mas é preciso considerar:

  • Limitação do dataset: coletado em condições controladas; performance em dados reais de hospitais pode ser inferior.
  • Falsos negativos existem: mesmo com Recall alto, qualquer caso não detectado representa risco à vida da paciente.
  • O modelo deve apresentar scores, não diagnósticos: "94% de probabilidade de malignidade" é mais útil e honesto do que "MALIGNO".
  • Explicabilidade é requisito clínico: o SHAP permite que o médico valide a lógica do modelo antes de considerar a predição.
  • Validação externa necessária: para uso clínico real, são necessárias validação em dados de outros hospitais, aprovação regulatória e integração com o sistema de prontuário existente.
O médico sempre tem a palavra final. O sistema proposto é uma ferramenta de triagem e priorização — não substitui a avaliação clínica especializada.

10. Como Executar

Pré-requisitos

  • Python 3.10+ (recomendado via Miniconda)
  • Conta no Kaggle para download dos datasets

Passos

  1. Instalar dependências:
    pip install -r requirements.txt
  2. Baixar dataset principal (Wisconsin):
    kaggle datasets download -d uciml/breast-cancer-wisconsin-data -p data/raw --unzip
  3. Baixar dataset extra (BUSI — necessário apenas para 04_cnn.ipynb):
    kaggle datasets download -d aryashah2k/breast-ultrasound-images-dataset -p data/raw/images --unzip
  4. Executar os notebooks em ordem:
    • notebooks/01_eda.ipynb — análise exploratória
    • notebooks/02_preprocessing.ipynb — pré-processamento
    • notebooks/03_modeling.ipynb — modelos ML e SHAP
    • notebooks/04_cnn.ipynb — [extra] CNN com Transfer Learning
